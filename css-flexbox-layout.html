Stage One: Understanding Flexbox

Introducing Flexbox

Flexbox offers the benefits of using block, inline-block and inline, without the quirks of using them. You can also change the direction, size and order regardless of the original order in HTML.

It's easier than ever to distribute space evenly to several elements on a page, or give more or less space to elements all within css flexbox rules, without the need to touch the html code.

Flexbox Basics and Terminology

The two more important elements in flexbox terminology are flexbox containers and flexbox items. The first step in using flexbox is creating a flex container; this will hold the flex items and sets the context for building the layout using the child item elements. A flex item can be any inline, block or inline-block item, and every direct child of a flexbox container is said to be a flexbox item. There can be any number of items inside a flexbox container.

For example: a ul would be the flex container and the li items are flex items, as they are children of the container parent in that relative context.

Creating a Flex Container

Before we can use any flexbox property, we need to create a flexbox container. This is done by assigning the flex value to the display property to the flexbox parent container (display: flex;)

Stage Two: Flexbox Properties

Controlling the Direction of Flex Items

Flex containers may take advantage of flexbox's direction, order, alignment, and sizing properties. It is important to know that some properties apply to flex containers only while others apply only to the flex items.

The flex-direction property applys only to the flex container; it will do nothing if applied to the items.

The row-reverse property for the flex-direction reverses the order of the items in the flex container.

By default, the browser tries to fit all items on one flexline. It will overflow content item, unless we code to wrap the remaining items, or increase height and or width dimensions.

Wrapping Flex Items

flex-wrap: wrap; when there isn't enough room on one line, the flex container will wrap the remaining items onto a new line in the direction of the flex axis. If we change the direction using flex-direction, that will be reflected in our browser.

flex-wrap: wrap; flex-direction: column; height: 280px; will cause the remaining items to wrap creating a new area of columns.

Distributing Space Inside a Flex Container

justify-content: Let's you control the position and alignment along the main-axis, and how space should be distributed in the flex-container. Justify-content will distribute the space that's available after the padding and margins have been accounted for.

Options like center, left, right, space-around, and space-between, let you control the position and alignment along the main-axis, and how space should be distributed in the flex-container. Justify-content will distribute the space that's available after the padding and margins have been accounted for. Space-around will give equal space to the left and right side of each element, and space between doesn't account for the edge space of the end elements, at least not in horizontal layout.

Also, when applying a margin: auto; to the first and second item, each item pulls itself in toward itself, taking up only the amount of space it needs and left-justifying itself.

Changing the Order of Flex Items

The order property applies to flex-items only, it will have no affect on parent containers.

Styling for .item-6 {order: -1; } will place item-6 just to the left of item-1; the default counting for the left-most item starts at 0, 0 being for item-1 in the html, translating to the flex-direction main and or cross access; if we used -2, then item-6 would still be just to the left of item-1 whose value is 0 by default. We can change these values for any item. This idea helps when styling for accessibility first, and then we can change the order as needed.

Growing Flex Items

.item {flex-grow: 0;} is default. Setting .item {flex-grow: 1;} causes the items to take up the full width of their parent container.

If we set .item-2 {flex-grow: 2;} then for each one width given to the other items, item two will gain 2, so it's a 2:1 ratio of width for item-2.

flex: 2; this will grow item-2 at 2:1 ratio over the other items set to flex-grow: 1;, this is shorthand for flex-grow.

It's good practice to use flex shorthand, and then target specific items using the other properties and values.

Smarter Layouts with Flex-Basis and Flex Grow

flex-basis gives even more control to items needing to have equal space. So we set the initial size we want the flex items to be, then flexbox equally distributes the space among the flex items. Until the items shrink below the flex-basis thresh-hold; after that the browser intelligently redistributes the space accounting for content differences.

Aligning Flex Items on the Cross Axis

Using align-items: stretch; will fill the space vertically inside the container according to the preset or default height of the container. Using align-items: flex-start; will cause the items to fill the container vertically up to their content only. align-items: flex-end; packs the items to the bottom of the cross axis, but only as high as their content.

Using align-items: center; will perfectly center all of the items along the cross axis, and we can choose to override any of these using the specific item property align-self. align-self can take the value flex-start, flex-end, stretch, or center

Creating a Three Column Layout with Flexbox

I have outlined this thoroughly in the notes in the styles.css file. I started by creating a parent div named " .row-3 and including 3 child divs with the class .flex-3 as a unifying class in addition to seperate class names for each .flex-column-1, 2 and 3 to specifically target any one of the child items. I included three columns of filler content. I also added an image to the first child, so that it was in turn the child of a child element. I gave the image a width of 40% of the parent column .flex-column-1.

Next I added an h3 heading to the first column, and then preceded to set its parent, flex-column-1 to a display: flex; in order to get the columns centered using align-items: center; Next I set a media query condition for 700px; my goal was to create a 2-column layout up to 1024px, and wrapping the third column to the bottom of the next line.

First I set the display: flex; property and value for the parent element in order to have a default row of three columns; next I set the flex-grow to 1 so that there would be equal width given to the three columns, but I also added a 50% value to the flex-grow shorthand; thus it was flex: 1 50%; Next I added a flex-wrap: wrap; to the parent .row-3 element so that the third column would breakdown to next line after the 2 previous columns took up 50% of the width apiece. Without adding the wrap value, all three columns would occupy the same row at once, creating a cramped feeling.

Next I targeted viewports of 1025px and greater. My goals were to have a three-column layout on one line, increase the size of the first column in ratio to the other two columns, and move the first column to the middle for page balance.

I set the flex-grow property of the flex-column-1 child item to 1.4 that of the other child item columns. I used the shorthand property. Next, to prevent the third column from wrapping in any case, I set the flex-basis: 0; so that wrapping would not take place until any item reached 0px, an impossible feat. Next I changed the order of the second column with the class .flex-column-2 to -1 so that it would shift to the left of the first column, whose order value was set at the default of 0. Now my first column appeared 1.4 times the size of the other two columns and it was positioned in the middle as desired.

Aligning Items to the Bottom of a Column

First I created three buttons, one for each column. I placed one each inside the .flex-3 divs. Next I gave the .flex-3 parent container to the buttons, the display: flex; Next, I gave the flex-direction: the value of column. Next I styled the .flex-button class with a margin-top: auto; to pin the buttons to the bottom of the flex container. Finally, to move the buttons to the left, I used the align-self: flex-start.

Creating a Sticky Footer with Flexbox

First, I set the body element's display property to flex, and flex-direction to column. Next, set min-height to 100vh. Lastly, target a wrapper, for me, it was .wrap and set the flex to 1.
