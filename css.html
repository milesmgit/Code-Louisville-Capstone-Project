Stage One: Getting Started with CSS Layout

Wrappers a.k.a. Containers

In web design, a wrapper (also known as a container) like div and section, is commonly used to center the layout on the page and keeps it from looking to wide or too narrow, depending on the device or viewport width.

Setting the div class="wrapper" { margin: 0 auto; } will center the child elements on the page.

Setting the div class="wrapper" width to 70% using a media query for screen larger than 768px width will keep the page from looking to spread out -- the narrower width will give the page some heft.

I set the h1 margin to 0 using the .name class because the body margin of 0 was collapsing into the h1 margin of .6em, because there wasn't any border, padding, or content between the parent body element and the h1 element, causing the h1 margin to dominate the area since it is the greater of the two values between body and h1.

Another way to ensure that the h1 margin doesn't collapse into the parent body element is to add some top padding to the .main-header class.

If two elements touch each other and each one has a margin set, the element with the smaller margin will collapse and the total margin between the two elements will be that of the larger margin value. Example. Block elements: upper element has a bottom margin of 30px, and the lower element has an upper margin of 20px. The 20px margin will collapse to 0, and there will be a total of 30px of margin between the two elements.

Creating a Sticky Footer

One way to ensure the footer stays at the bottom of a webpage is to add a min-height to a wrapper which encloses all elements except for the footer element. vh = viewport height. 1vh = 1% of the height of the viewport, so 100vh is 100% of the height of the viewport. We can use the developer tools to determine our footer's height (89px in this case, and if our footer height changes, we need to recalculate and adjust), and then we can use the calc function to do the math of: min-height: calc(100vh - 89px); This will place our footer precisely where it needs to be.

Stage Two: Controlling Layout with CSS Display Modes

Positioning Elements Side-By-Side with Inline Display

UL (Unordered Lists) li (list items), divs, paragraphs and headings are block elements by default, and as such, have line breaks before and after them.

Images, links and span tags are inline and do not contain line breaks.

When we use inline display elements, the browser does not apply any width and height properties, or top and bottom margin settings; it will only accept left and right margin settings and padding settings.

Using Inline-Block to Set Width, Height, Top and Bottom Margin, and Padding

The advantage to using inline-block over inline is that you can add top and bottom margin and width and height to the element.

By default block elements take up the width of their container whether or not they need the space for content; inline-block will be as wide as the content before additional css.

Setting a UL (unordered list) or ol (ordered list) to inline-block will cause all of the list items to behave as one block by default. They will stay together during resizing.

Removing Gaps Between Inline and Inline-Block Elements

Inline and inline-block elements have spaces between them just like the words of a paragraph; to remove them, you can add negative right-margin. You can also group list-items back to back without spaces <li>content</li><li>content, or not add a closing li tag.

An elements font-size will affect the gaps between inline and inline-block elements; the larger the font-size, the greater the gap.

Stage Three: Page Layout with the Float Property

How Floats Work

Floats take the element out of the normal flow of the page: You can have text wrap around an image by floating the image to the right or left.

Float accepts three values: left, right, or none.

Using margin on the paragraph element will not have any effect on the img because we have taken the img out of the normal flow of the page. Apply the margins to the img instead.

Stage Four: Positioning Page Content

How Absolute Positioning Works

position: absolute; will take the element out of the normal flow and place it in the top left corner of its parent container. They are neither affected by or affect other elements on the page.

position: absolute; top: 0; left: 0; top: and left: are called position offsets. Percentage, em, and pixels are data types that may be used for the offsets. The position of the elements will be relative to their parent container. They will be placed in relation to the first parent that has a relative position; if none has a relative position, then the browser window will be the relative parent.

Positioning Elements Relative to Other Elements

Suppose we have a header element and then we place list items inside the header div container. Then we set the list items to absolute position. They will be placed relative to the browser viewport since we didn't give the header container a relative position value. If we then give the header element a position value of relative (position: relative;), which is the closest parent of the list items, the list items will move around inside the header element depending on the data type values we pass to them.

Fixed Positioning

Fixed positioning is a form of absolute positioning that places the element in relation to the browser window instead of a parent element.

Unlike absolute positioning, fixed positioning will stay fixed to one spot on the browser window; that is the user never loses sight of the nav menu, because it stays fixed to an area on the viewport.

Because the position: fixed of my main-header cuts off the remainder of my body by the amount in the chrome developer tools for my header height, which happens to be 194px, I add a padding-top to the body element for that amount. This restores my page proportions.

How Z-Index Works

When relative, absolute and or fixed elements are used, they may end up occupying the same space, so we will override this behavior using the z axis, or 3d space stacking order so to speak (to the front, or back).

Fixed, relative and or absolute styled position elements that appear later in the code will roll over those that come before it, and by later, I mean lower on the html page.

We can adjust the stacking order using the z-index's z-axis. An element with a higher z-index will apear over those with lower z-index: example 20 z-index will roll over a 10 z-index.

Positioned elements have a default z-index of 0, so we will need to make a few changes. The code is: z-index: 1000 to be safe in case I add more position elements later;

Position elements with a z-index are in a stacking order relative to their parent element.
