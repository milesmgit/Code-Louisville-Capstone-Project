Stage One: Responsive Theory

What is Responsive Web Design

Responsive Web Design consists of fluid grids, fluid images, and media queries.

It is important to think in terms of ratios, rems, ems and or percentages when designing elements for various screens so that the elements can respond intelligently and adjust accordingly.

Elements that use percentages apply those percentages to their parent container's dimensions.

Stage Two: Media Queries

Media Query Review

Media Queries are CSS rules that use CSS code to style a page only when certain conditions are met; these conditions are known as Media Features, and are usually based on a screen's minimum-width.

https://developer/mozilla.org/en-US/docs/Web/Guide/CSS/Media_queries is a good place ot learn about media features.

Orientation is a handy media feature for detecting whether or not a phone is displaying content using landscape or portrait mode.

Example media query targeting landscape orientation to prevent unecessary scrolling due to large headers taking up too much height: @media only screen and (min-width, 768px), only screen and (min-width: 700px) and (orientation: landscape) { apply css rules as needed here; }

Understanding Breakpoints

Breakpoints can be thought of in terms of width; it's that point in a screen's width where the layout no longer works and the screen elements breakdown and go all wacky. Generally, any media queries that adjusts the layout based upon the width can be thought of as a breakpoint.

Creating breakpoints based upon certain thresholds that affect the appeal of our one-column, mobile-first approach is a more sustainable technique than building for device screen-sizes.

Breakpoints generally don't add a time cost to the site, though they do add to the overall complexity of the code.

Responsive Design Patterns
